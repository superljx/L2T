# 🔍 倒计时识别失败调试指南

## 问题分析

`test_timer.py` 能识别出"14秒"，但 `main.py` 实际运行时识别失败。

### 可能原因

1. **截图区域不同** - main.py捕获的区域与example.png不一致
2. **实时画面有干扰** - 游戏中有其他UI元素、特效
3. **分辨率缩放** - 游戏窗口大小与截图不同
4. **时机问题** - 捕获时数字正在变化或有动画

---

## 🔧 调试步骤

### 步骤1：启用倒计时ROI保存

编辑 `config.py`：
```python
DEBUG_MODE = True
DEBUG_SAVE_TIMER_ROI = True  # 保存倒计时识别区域
```

### 步骤2：运行程序并触发检测

```bash
python main.py
```

游戏中阵亡后，程序会保存 `debug_screenshots/timer_roi_HHMMSS.png`

### 步骤3：对比两个图片

```
example.png              vs    debug_screenshots/timer_roi_HHMMSS.png
(test_timer能识别)             (main.py实际捕获的)
```

对比内容：
- ✅ 区域位置是否相同？
- ✅ 数字是否清晰？
- ✅ 是否有其他干扰元素？
- ✅ 分辨率是否一致？

### 步骤4：根据差异调整

#### 情况A：区域位置不对

调整 `detector.py` 中的区域参数：
```python
# 在 extract_respawn_timer 函数中
x1 = int(w * 0.35)  # 左边界（0.35 = 35%）
x2 = int(w * 0.65)  # 右边界（0.65 = 65%）
y1 = int(h * 0.45)  # 上边界（0.45 = 45%）
y2 = int(h * 0.70)  # 下边界（0.70 = 70%）
```

根据实际情况调整这些百分比。

#### 情况B：分辨率太小

游戏分辨率建议：
- 最佳：1920×1080
- 最低：1280×720

#### 情况C：有干扰元素

启用更严格的金色过滤：
```python
# 在 preprocess_for_gold_numbers 中调整HSV范围
lower_gold = np.array([15, 150, 150])  # 更严格
upper_gold = np.array([35, 255, 255])
```

## 📊 常见情况及解决方案

### 情况1：ROI完全捕获错了

**症状**：保存的ROI图片里根本没有倒计时数字

**解决**：
1. 检查游戏是否全屏独占（改为窗口化全屏）
2. 检查是否多显示器（指定主显示器区域）
3. 调整ROI区域百分比

### 情况2：数字很模糊

**症状**：ROI图片里的数字看不清

**解决**：
1. 提高游戏分辨率
2. 关闭游戏内的模糊效果
3. 确保UI缩放为100%

### 情况3：识别到错误的数字

**症状**：识别成其他数字（如4而不是14）

**解决**：
已优化：程序会尝试20种组合，取最常见的结果
如果仍然错误，启用DEBUG_MODE查看所有尝试结果

### 情况4：有其他UI元素干扰

**症状**：ROI区域有其他文字、图标

**解决**：
1. 缩小ROI区域，只包含数字
2. 使用更严格的颜色过滤
3. 等待UI动画结束（增加CONFIRM_FRAMES）

## 💡 终极解决方案

如果调试后仍然无法识别，可以：

### 方案A：手动标注区域

1. 使用截图工具测量倒计时数字的精确像素位置
2. 直接在代码中使用绝对坐标（而不是百分比）

### 方案B：降低要求

```python
# config.py
CONFIRM_FRAMES = 1  # 减少确认要求，更容易触发
```

### 方案C：使用模板匹配作为备用

如果OCR实在不行，可以考虑：
1. 提前准备0-9数字的模板图片
2. 使用模板匹配识别数字
