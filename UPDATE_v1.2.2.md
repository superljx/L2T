# 🔧 v1.2.2 更新说明 - 修复倒计时期间重复触发

## 🐛 修复的Bug

### Bug: 倒计时期间冷却结束后重复触发切换

**症状**：
```
18:13:36 设置复活倒计时: 53秒
18:14:04 触发自动切换流程  ← 倒计时还剩29秒就又触发了！
```

**原因分析**：
1. 检测到阵亡，设置倒计时53秒
2. 切换到Edge播放视频
3. 同时触发冷却时间15秒
4. 15秒后冷却结束
5. 程序仍能检测到死亡状态（通过OCR或其他方式）
6. 再次触发切换流程 ❌

**根本问题**：
- 冷却时间(15秒) < 复活倒计时(53秒)
- 冷却结束后，没有检查是否正在等待复活
- 导致重复触发切换

---

## ✅ 修复方案

### 修改1: 增加倒计时状态检查

```python
# 之前：只检查冷却时间
if state == "dead":
    if self.cooldown.can_trigger():
        self.handle_dead_state()  # ❌ 重复触发

# 现在：先检查是否在倒计时中
if state == "dead":
    if self.is_waiting_respawn:
        # 正在等待复活，不要重复触发
        pass
    elif self.cooldown.can_trigger():
        self.handle_dead_state()  # ✅ 只在合适时触发
```

### 修改2: 切回游戏后重置冷却

```python
# 倒计时结束切回游戏后
self.switch_back_to_game()
self.is_waiting_respawn = False
self.cooldown.trigger()  # 重置冷却，为下次死亡做准备
```

## 🎯 现在的正确流程

```
1. 检测到阵亡，倒计时53秒
   ↓
2. 触发切换，播放视频
   ↓
3. 冷却时间15秒开始计时
   ↓
4. 15秒后冷却结束
   ↓
5. 检查：is_waiting_respawn = True
   → 跳过，不重复触发 ✅
   ↓
6. 继续等待倒计时（53秒）
   ↓
7. 倒计时剩余2秒时切回游戏
   ↓
8. 重置状态，准备下次检测
```

## 💡 技术细节

### 状态标志

- `is_waiting_respawn`: 是否正在等待复活
- `respawn_timer_start`: 倒计时开始时间
- `respawn_duration`: 倒计时总时长

### 优先级

```
1. 正在等待复活？ → 跳过检测
2. 冷却时间未结束？ → 跳过触发
3. 检测到阵亡？ → 触发切换
```
